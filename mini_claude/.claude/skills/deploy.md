# /deploy — 部署助手

请执行以下步骤：

## 1. 预检查
- 运行 `git status` 确认工作区干净（无未提交变更）
- 运行测试确保全部通过
- 检查是否在正确的分支上

## 2. 判断部署目标
根据项目类型选择部署方式：

### Docker 部署（mini_claw）
```bash
cd deploy/
# 检查 .env 是否存在
# 执行 bash start.sh 或 bash update.sh（如果已运行）
```

### PyPI / pip 部署
```bash
# 更新版本号
# python -m build
# twine upload dist/*
```

### 通用流程
1. 确认部署环境和配置
2. 构建产物
3. 执行部署命令
4. 验证部署结果（健康检查）

## 3. 部署后验证
- 检查服务健康状态
- 输出部署摘要（版本、时间、环境）

注意：
- 所有破坏性操作前必须向用户确认
- 不要自动推送到远程仓库，除非用户明确要求
- 记录部署日志
