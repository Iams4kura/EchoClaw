# /test — 智能测试

根据参数决定行为：

## 无参数：运行现有测试
1. 检测项目测试框架（pytest / unittest / jest 等）
2. 运行全部测试：`pytest` 或对应命令
3. 如果有失败，分析失败原因并建议修复

## 有参数（文件路径）：为该文件生成测试
1. 读取目标文件，理解每个公开函数/类的职责
2. 确定测试文件位置（遵循项目现有的测试目录结构）
3. 生成测试用例，覆盖：
   - 正常路径 (happy path)
   - 边界条件 (edge cases)
   - 错误处理 (error cases)
4. 编写测试代码，遵循项目现有的测试风格：
   - 使用项目已有的 fixture 和 helper
   - 异步代码用 `pytest-asyncio`
   - mock 外部依赖（网络、数据库、文件系统）
5. 运行生成的测试确保通过

测试命名规范：`test_{功能}_{场景}_{预期结果}`

注意：
- 不要写只测试 mock 自身的无意义测试
- 优先测试业务逻辑，不测试框架行为
- 本项目用 pytest，asyncio_mode = "auto"
